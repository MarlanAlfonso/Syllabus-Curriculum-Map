# Wireframe KM Annotations

## Screen 1: Home / Dashboard
The dashboard serves as the entry point to the knowledge map. It orients the user
to the system's purpose — navigating course relationships — and provides direct
access to the two primary knowledge views: the course list (retrieval) and the
curriculum map (visualization of knowledge flow).

## Screen 2: Course List Page
The tabular layout supports knowledge retrieval — users can scan, sort, and locate
specific courses (knowledge nodes) efficiently. The Edit button exposes prerequisite
links for modification. The Disable button soft-removes a node without deleting
knowledge links.

## Screen 3: Add / Edit Course Form
The prerequisite multi-select is the key knowledge linking interface — it creates
edges between knowledge nodes. The skills tag input builds the knowledge taxonomy.
Together they encode both relational structure (links) and descriptive classification
(taxonomy) into each course record.

## Screen 4: Curriculum Map View
The directed graph makes knowledge flow visible — arrows show which knowledge must
exist before new knowledge can be built. The filter controls allow users to isolate
specific knowledge domains. Isolated nodes (no prerequisites, no dependents)
represent standalone knowledge units or potential curriculum gaps.
